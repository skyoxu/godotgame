# Workflow Rule Feedback Record

## 1. Rule Identity

- Rule name: extract timeout retry boost
- Workflow family: `5.1`
- Source repository: `skyoxu/newrouge`
- Source branch: `main`
- Source commit: `444002a047f8739c7ba34c090af6d1abc3558b46`
- Date: `2026-04-19`

## 2. Problem Statement

- Repeated workflow 5.1 runs showed that `extract` and `align` could hit timeout first, even when the task itself was still worth processing.
- The old behavior forced operators to rerun the whole task or whole batch with larger budgets, which wasted deterministic work and made batch recovery noisy.

## 3. Observed Symptoms

- Operator-visible symptom: the same task failed early on `extract` or `align`, then succeeded after a targeted timeout increase.
- Cost symptom: repeated full-step reruns paid for downstream steps that had not yet become relevant.
- Quality symptom: no direct false-green risk, but repeated broad reruns obscured the real bottleneck.
- Recovery symptom: summary-level guidance stayed too coarse without a step-local timeout retry path.

## 4. Artifact Evidence

- `summary.json`: repeated failure aggregation and retry-worthy timeout patterns
- `report.md`: operator-visible failure recap for the affected run
- per-step logs under `logs/ci/**`: `extract` / `align` timeouts before downstream steps
- implementation evidence:
  - `scripts/python/run_single_task_light_lane.py`
  - `_RETRYABLE_TIMEOUT_STEPS = {"extract", "align"}`
  - `_RETRY_TIMEOUT_BOOST_SEC = 240`
  - `_run_step_with_retry(...)`

## 5. Failure / Waste Pattern

- Failure family: timeout
- Trigger condition: `extract` or `align` exits with `rc=124`
- Repeated how many times: enough to justify built-in retry support in the business repo
- Affected lane or step: workflow `5.1`, step-local `extract` / `align`

## 6. Proposed Reusable Rule

- Proposed rule in one sentence: when workflow 5.1 times out in `extract` or `align`, retry only that step once with a larger timeout instead of rerunning the whole task immediately.
- Proposed template target:
  - classifier/helper
  - producer script
  - workflow doc
- Suggested files:
  - `scripts/python/run_single_task_light_lane.py`
  - `workflow.md`
  - `docs/workflows/stable-public-entrypoints.md`

- Why: this is a workflow-agnostic cost-control rule that does not weaken deterministic gates and applies to the same template entrypoint used by multiple repositories.
