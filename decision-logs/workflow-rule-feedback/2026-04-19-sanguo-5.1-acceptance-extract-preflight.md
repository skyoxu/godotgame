## 2. Problem Statement

- Long workflow 5.1 runs in the business repository showed that some tasks were doomed before expensive extraction started because their acceptance / refs / task-view prerequisites were incomplete.
- Without an early deterministic guard, the wrapper paid for LLM extract work and downstream semantic steps before discovering a deterministic input problem.

## 3. Observed Symptoms

- Operator-visible symptom: full 5.1 batches spent time on tasks that later failed because required acceptance evidence was missing or malformed.
- Cost symptom: unnecessary extract / align / semantic-gate cost on tasks that should have been stopped before LLM work.
- Quality symptom: no quality loss from early guard; it stops invalid inputs before generation.
- Recovery symptom: failures became easier to classify because deterministic preflight failures were separated from model failures.

## 5. Failure / Waste Pattern

## 6. Proposed Reusable Rule
